# AKTIVITÄTSMESSER

A 30-Tage-Challenge activity tracker PWA for Ringana partners. Built on top of the project's Ringana design system (`../colors_and_type.css`).

## Open it

Open `index.html`. Mobile-first, runs inside an iOS frame.

## What it does

Track daily business activities — visibility posts, contact conversations, follow-ups, sales, mindset work — and watch points accumulate toward your daily goal. All 22 activities and point values are taken straight from the spec.

- **Onboarding** (3 steps): name → daily-goal slider → "so funktioniert's"
- **Heute** (Today): personal greeting, day navigator (any day from 1.1.2026 onward), progress ring, week/month totals, Sichtbarkeit + Mindset quick-toggles, contact list, bottom sheet with 14 activity chips per contact, inline rename, "+ Kontakt hinzufügen"
- **Kalender** (Calendar / Statistics): month grid with per-day point numbers, met-goal days highlighted in olive, monthly summary stats (Summe / Ziel erreicht / Schnitt), and a full **Übersicht** breakdown of the currently selected day
- **Einstellungen** (Settings): edit name, change default daily goal, per-day goal override, anonymization toggle (renames all contacts to *Person 1, Person 2 …*), full reset

State persists in `localStorage` under `aktivitaetsmesser:v1`. Refresh and it remembers everything. Nothing is uploaded.

## File structure

| File | Role |
|---|---|
| `index.html` | **Single self-contained bundle** — all CSS and JSX are inlined. This is what runs. |
| `manifest.webmanifest` | PWA manifest. (Icons `icon-192.png` / `icon-512.png` still to be supplied.) |
| `app.css` | App styles (extends `../colors_and_type.css`). Edit this and re-bundle. |
| `data.jsx` | Activity catalog (22 activities × 5 groups), date helpers, localStorage layer, point math |
| `ui.jsx` | Primitives: `Icon`, `Btn`, `ProgressRing`, `Sheet`, `Chip`, `DailyToggle`, `ContactRow`, `TabBar`, `Switch` |
| `onboarding.jsx` | 3-step welcome flow |
| `home.jsx` | Today screen + contact bottom sheet |
| `calendar.jsx` | Month grid + day breakdown |
| `settings.jsx` | Profile, goals, privacy, reset |
| `app.jsx` | App container with tabs + persisted state |
| `ios-frame.jsx` | Phone bezel (from project starter) |

The split files are the **canonical source**. `index.html` is generated by concatenating them into one document because the project's cross-file serving was returning 401s during development — inlining everything sidesteps that. **If you edit any of the `.jsx` or `app.css` files, re-bundle with the script in `BUNDLE.md`** (or just ask the AI to do it).

## Color & type

Inherits everything from `../colors_and_type.css`. App-specific overrides (the olive-green app primary, category dot colors) live in `app.css` under `:root`.

## Known gaps / next steps

- **Icons:** Not yet exported as `icon-192.png` / `icon-512.png` for the manifest. The manifest references them but they don't exist on disk. Drop two PNGs in to make the PWA installable.
- **Offline:** No service worker yet. The PWA manifest is in place, but full offline support needs a `sw.js` that caches `index.html` + the Google Fonts + Lucide CDN.
- **Touch-and-hold to rename a contact:** Only tap-pencil works in the bottom sheet. Could add long-press in the list row too.
- **Streak tracking:** Not in the spec, intentionally omitted. The feel was meant to stay quiet, not gamified.
